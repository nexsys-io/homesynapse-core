# Spike Directory

Everything in this directory is **throwaway code**.

## Rules

1. Spike code exists to answer a specific question or validate a technical approach.
2. Findings from spikes are documented in the `homesynapse-core-docs` repository,
   not here.
3. Spike code must never be promoted to production. If a spike validates an
   approach, the production implementation is written fresh following the design
   documents.
4. Spike code does not need to follow coding standards, but must not be committed
   to `main` without a clear label that it is a spike.
